/speckit.constitution

Principi fondamentali del progetto

/speckit.constitution

API REST per e-commerce:
- Node.js 20 + TypeScript strict
- Test coverage >85% (Jest + Supertest)
- Rate limiting 100 req/min per user
- Response time p95 < 200ms
- OpenAPI 3.0 spec completa
- Zero downtime deployments

/speckit.specify

COSA vuoi costruire (user stories)

/speckit.specify

Endpoint gestione ordini:
- POST /orders: crea ordine da carrello
- GET /orders/:id: dettagli ordine
- PATCH /orders/:id/status: aggiorna stato
- GET /orders: lista con paginazione + filtri

Auth: JWT (vedi docs/auth.md)
Pagamenti: Stripe webhook per conferma
Errori: formato standard con code + message

/speckit.clarify

Risolve ambiguità (opzionale)

/speckit.clarify

L'AI chiede:
- "Paginazione: cursor-based o offset?"
- "Status ordine: quali transizioni valide?"
- "Webhook Stripe: retry policy?"
- "Race condition su stock: come gestire?"
- "Rollback su payment failure?"

/speckit.plan

COME implementare (tech stack)

/speckit.plan

Stack:
- Express + TypeScript + Prisma
- PostgreSQL con transazioni SERIALIZABLE
- Redis per rate limiting + cache
- Bull queue per async operations
- Stripe SDK per payments

Pattern:
- Repository pattern per DB
- Middleware chain per validation/auth
- Event emitter per webhooks

/speckit.tasks → /speckit.implement

Task list → esecuzione automatica

/speckit.tasks genera:
1. Prisma schema: Order, OrderItem, Payment
2. DTO + validation schemas (Zod)
3. OrderRepository con transactions
4. OrderService + business logic
5. Express routes + middleware
6. Stripe webhook handler
7. Unit + integration tests
8. OpenAPI spec generation

/speckit.implement → esegue tutto
